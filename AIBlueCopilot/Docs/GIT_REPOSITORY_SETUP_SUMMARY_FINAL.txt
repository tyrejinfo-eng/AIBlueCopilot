## ?? Pro Tips for Your Team

### Prevent Common Mistakes

```
? Never:
- Push directly to main (use PRs)
- Commit to your branch without testing
- Leave commits unreviewed for days
- Use vague commit messages ("fixed stuff")
- Commit API keys or passwords

? Always:
- Pull before you start work
- Push frequently (once per feature)
- Ask for code review within 24 hours
- Use clear commit messages
- Check .gitignore before committing large files
```

### Keep Team Happy

```
? Code Review Best Practices:
- Review PRs within 24 hours
- Leave constructive comments
- Approve quickly if code is good
- Ask questions if unclear

? Communication:
- Use GitHub Issues for tasks (not email)
- Use PR comments for code discussion
- Use Discord/Slack for urgent items
- Weekly sync call (30 min)

? Workflow:
- One feature per branch
- Keep commits small and focused
- Write clear PR descriptions
- Delete branch after merge
```
