## ?? First Time Setup (Do Once)

```powershell
# 1. Configure Git
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# 2. Clone repository
cd Documents
git clone https://github.com/YOUR_TEAM/AIBlueCopilot.git
cd AIBlueCopilot

# 3. Done! You're ready to work
```

## ?? Daily Workflow

```powershell
# Morning: Get latest code
git checkout develop
git pull origin develop

# During day: Create feature branch
git checkout -b feature/what-you-are-building

# Make changes in Visual Studio...

# End of day: Save your work
git add .
git commit -m "feat: Description of what you did"
git push origin feature/what-you-are-building
```

---

## ?? Merge Your Changes (Push to Team)

1. Go to GitHub in browser
2. You'll see "Compare & pull request" button
3. Click it
4. Add description of changes
5. Click "Create Pull Request"
6. Wait for teammate to review ?
7. Teammate clicks "Approve"
8. You click "Merge Pull Request"
9. Delete the branch (button appears)

---

## ? Reviewing Teammate's Code

1. Go to GitHub "Pull Requests" tab
2. Click PR to view
3. Click "Files changed" to see code
4. Hover lines to add comments
5. Click "Approve" when good
6. Don't click merge - they will

## ?? Quick Fixes for Mistakes

| Problem | Fix |
|---------|-----|
| Want latest code | `git pull origin develop` |
| Made local change you want to undo | `git checkout .` |
| Forgot to create branch | `git checkout -b feature/name` |
| Committed but not pushed yet | `git reset --soft HEAD~1` |
| Want to see what you changed | `git diff` |
| Want to see commit history | `git log --oneline` |
| Need to switch branches | `git checkout branch-name` |
| Want to list all branches | `git branch -a` |


## ?? Basic Commands

```powershell
# See what changed
git status

# See detailed changes
git diff

# Stage changes for commit
git add .

# Commit changes
git commit -m "message"

# Push to GitHub
git push origin branch-name

# Get latest from team
git pull origin develop

# Create new branch
git checkout -b feature/name

# Switch to existing branch
git checkout develop

# Delete local branch
git branch -d branch-name

# List all branches
git branch -a
```


## ?? One Minute Lessons

### What is a Branch?
Independent copy of code where you work without affecting others.

### What is a Commit?
Saving your changes with a description of what you did.

### What is a Push?
Uploading your commits to GitHub so team can see them.

### What is a Pull Request?
Asking team to review your code before merging to main branch.

### What is a Merge?
Combining your branch changes back into develop/main.

---

## ?? Best Practices

? **Do:**
- Pull before you start work
- Create new branch for each feature
- Push frequently (daily)
- Write clear commit messages
- Ask for code review
- Keep commits small and focused

? **Don't:**
- Commit and push without testing
- Use vague commit messages
- Leave PRs unreviewed
- Work directly on develop/main
- Commit large binary files
- Commit secrets or passwords

## ?? Troubleshooting Decision Tree

```
Did you mess up?
?
?? Only locally (not pushed yet)? ? git checkout .
?
?? Pushed to wrong branch? ? Create new PR to correct branch
?
?? Committed something bad? ? Tell project lead immediately
?
?? Need to undo latest commit? ? git reset --soft HEAD~1
?
?? Still confused? ? Ask team on Discord/Slack
```
